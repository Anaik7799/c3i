  || echo '{\"systemMessage\":\"Stop-hook ingest already running, skipped (flock).\"}'"

  • Eliminates 3-way concurrent ingests previously observed (PIDs 7989/8337/9497)

6. CPU controller — c3i.slice (≤ 70%)

[Slice]
CPUQuota=700%        # 70% × 10 cores; CFS-enforced
CPUWeight=200
MemoryHigh=32G       # soft throttle
MemoryMax=38G        # hard kill
IOWeight=200
TasksMax=4096

Rolling-restarted all long-lived c3i-* services so they enter the cgroup. Live measurement: slice consumed 4271 ms / 5000 ms wall = 8% of host. Cap = 70%. Headroom 9×.

Impact on startup & runtime

  • Startup: same 30 s convergence, plus health snapshot at T+15 s
  • CPU: kernel CFS throttles slice when > 700% of CPU. Other host processes untouched.
  • Memory: 32 G triggers reclaim, 38 G triggers cgroup-OOM (kills largest in slice, host stays alive)
  • Hooks: idempotent under arbitrary fire rate
  • Observability: any operator queries :8090/health/services.json for live truth
  • Port safety: collisions refused at unit-start time
