5. Gap 5 — Topic algebra for health publishing

Zenoh key expressions form a tree; selectors support glob (*) and double-glob (**).

indrajaal/l2/health/
├── snapshot                        ← full JSON every 30 s (snapshot publisher)
├── slice/
│   ├── memory_bytes
│   └── cpu_nsec
└── {unit}/
    ├── state                        ← active|inactive|failed|activating
    ├── memory_bytes
    └── cpu_nsec

Subscribers can use narrow selectors for targeted observation:

z.subscribe("indrajaal/l2/health/c3i-zenoh-router/state", on_zenoh_state_change)
z.subscribe("indrajaal/l2/health/*/memory_bytes",          on_any_memory_change)

Mutual information I(unit_state; system_health) is highest at the per-unit topic; subscribers avoid 30 KB JSON parse per snapshot if they only care about one service.

6. Gap 6 — Lyapunov hysteresis controller for memory pressure

stable iff dV/dt × Tr < Δ ⇒ growth rate < 0.9 units/s
