5-Why Root Cause

  1. Why prod read legacy? Vault contained 1 of 9 secrets
  2. Why empty? No operator migration tooling
  3. Why no tooling? Gleam vault.put stubbed; Rust had no write API
  4. Why Gleam stubbed? Slice B depends on rusty_vault_nif put_secret/4
  5. Root: SC-VAULT-009 envisioned single Gleam writer, never wired, no parallel Rust writer authorized — read-side compliant, write-side unimplementable, silent prod fallback

Architectural Trade-Off

SC-VAULT-009 single-writer mandate

Pass-1 added a Rust writer that produces envelopes interop-compatible with the Gleam-NIF writer. Both can coexist iff they share the KEK file + AES-256-GCM wire format.

The contract

  • Same C3I_VAULT_KEK_PATH sidecar (32 raw bytes)
  • Same wire envelope: nonce(12) || ciphertext+tag
  • Same kv_entries + audit_log schema

Documented inline in vault.rs::encrypt_envelope. Future passes can deprecate the Rust writer if Slice B Gleam writer ships.
