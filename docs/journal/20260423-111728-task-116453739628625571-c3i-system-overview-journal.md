## 1. Executive summary

C3I (**Constitutional Cognitive Cyber Infrastructure**) is a SIL-6-grade, constitutionally-governed
multi-agent operating platform. It unifies four things most stacks keep separate:

1. a **biomorphic 25-agent swarm** organised in a 3-layer Viable-System-Model hierarchy,
2. a **fractal holonic architecture** in which the same invariants (Ψ-0..Ψ-5) repeat at eight self-similar
   layers **L0 Constitutional → L7 Federation**,
3. a **triple-interface UI** (Gleam Lustre SSR + Wisp REST + TUI split-screen) that share one typed domain
   model and publish every state change as an OTel span on a Zenoh pub/sub mesh,
4. a **Pi symbiosis bridge** that gives any LLM agent (Claude / Gemini / local) safe, gated, circuit-
   broken access to the whole system via MCP-over-Zenoh.

Everything is **audited, gated, and verifiable**: every destructive tool call is arbitrated by the Guardian
safety kernel, every test run must clear math gates (Shannon entropy H ≥ 2.5, CCM ≥ 0.90, ITQS ≥ 0.85,
D_EA ≤ 10 %), and every artefact is ingested into the Zettelkasten (FTS5 + fractal tags) for future recall.

## 2. Why it exists

C3I exists to turn AI agent fleets into a **governable control system** rather than a loose collection of
chat wrappers. The design goals are:

- **Constitutional safety** — no agent can bypass Ψ invariants or the Guardian L0 gate.
- **Observability by construction** — if it happens, it is on the Zenoh bus and in an OTel span.
- **Triple-interface parity** — every feature must ship simultaneously as Lustre page, Wisp endpoint and
  TUI view, with *one* shared type module. No drift.
- **Mathematical coverage** — tests are measured by information theory, not line count.
- **Zero-waste (Muda)** — every artefact (diagram, journal, span, task row) has a downstream consumer.

## 7. AG-UI and A2UI protocols

- **AG-UI** is a 32-event protocol: `RunStarted`, `StepStarted`, `ToolCall`, `ToolResult`, `HITLRequested`,
  `RunFinished`, `RunError`, … Every Pi emission and every UI action maps to one AG-UI event, is
  published on Zenoh under `indrajaal/agui/**`, and is validated by `testing/zenoh_test_observer.gleam`.
- **A2UI** is a 16-component allowlisted catalog (`a2ui/catalog.gleam`) with renderer parity across
  HTML / JSON / ANSI. Agents may *propose* UI; the validator either accepts or rejects. No arbitrary
  markup reaches the browser.

## 9. Pi symbiosis

Pi is the bridge that lets an external LLM act as a first-class C3I agent without bypassing safety:

- `pi_agent.gleam`, `pi_tools.gleam`, `pi_session.gleam`, `pi_provider.gleam`, `pi_zenoh.gleam`
- Every tool call routes through Guardian L0 when destructive/privileged
- Every LLM call is wrapped in a circuit breaker + optional hedged race
- Session state persists in `Smriti.db` (JSONL allowed locally, Smriti required in prod)
- Model registry stays in sync with the C3I resolver (L5)
- MCP tool traffic is carried as **MCP-over-Zenoh** (MoZ) with correlation IDs

`pi-evolution-verification.md` requires every feature to run the Pi integration tests, publish spans,
update the dashboard and ingest the journal into ZK.
