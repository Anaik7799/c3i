Decision Framework

Is the value a pre-serialized JSON string from storage? ┌─ YES → serdes_json (object_raw, encode_event) └─ NO → Are all values Gleam-typed (String, Int, Bool)? ┌─ YES → gleam/json (json.object, json.string) └─ NO → Is it query/manipulation (pointer, merge, diff)? ┌─ YES → serdes_json └─ NO → gleam/json (default)

Key Insight

gleam/json is for CONSTRUCTION — building JSON from typed Gleam values.

serdes_json is for COMPOSITION — assembling JSON from pre-existing fragments.

Both together — gleam/json builds the skeleton, serdes_json fills in the raw protocol blobs.
