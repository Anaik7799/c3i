### 3.3 FINDING-A closure (RPN=900)

**Problem** (per ultra-pass-4 ZK): `zenoh_otel.publish_span(session, span)` required the caller to own a zenoh Session, so no UI page actually emitted spans.

**Fix:** added **two** session-less convenience functions in `lib/cepaf_gleam/src/cepaf_gleam/ui/zenoh_otel.gleam`:

```gleam
pub fn publish(span: OtelSpan) -> Result(Nil, String) {
  let topic = otel_topic(span.page, span.element)
  let payload = json.to_string(span_to_json(span))
  case client.put_nif(topic, payload) {
    Ok(_) -> Ok(Nil)
    Error(msg) -> Error(msg)
  }
}

pub fn emit_span(page, element, phase, attributes) -> Result(Nil, String) {
  publish(new_span(page, element, phase, attributes))
}
```

Both wrap `client.put_nif` (already NIF-bound in `zenoh/client.gleam:13`). No caller needs to thread a `client.Session` anymore. Build: `Compiled in 0.23s` (no new warnings).

## 6. Decisions & Rationale

1. **Session-less publish** (FINDING-A) — chose `put_nif` over threading `client.Session` because (a) it eliminates a refactor of 57 Lustre pages, (b) the NIF layer already owns an ambient Zenoh session lifecycle, (c) pattern matches `pi_zenoh.gleam` precedent.
2. **Relaunch rather than surgical restart** — after the `priv/native` fix, idempotent `sa-up launch` relaunches only the 4 missing replicas; safer than per-container surgery.
3. **PID-file unlock by delete** — timestamp_daemon will rewrite on boot; no data loss because `timestamp-state.json` is separate.
4. **Child-task granularity** — 15 children, one per RPN row from ultra-pass-4, preserves fractal self-similarity across tasks↔findings.
