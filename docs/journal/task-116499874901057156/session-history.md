### Section A — deep dive on top 12 libraries

#### A1. frunk 0.4.4
**What it exposes**: `HList!`, `Coproduct`, `Generic`/`LabelledGeneric` (derive macro), `Validated<HList, E>` (applicative error accumulation), `Semigroup`/`Monoid` (with blanket impls for Vec/HashMap/Option/String/numerics), `Poly`/`Func` (type-level dispatch).
**Where it lands in C3I**: `db.rs` `validate_priority`/`validate_status` chains → `Validated<HList!(Priority, Status, Owner), Vec<IgnitionError>>` so SC-VALUE-GUARD reports ALL violations per row. `cortex.rs` intent classifier → `Coproduct!(SimpleQuery, ComplexInfer, ToolCall, Refusal)`. `pii.rs` `Monoid for ScrubReport`. `fmea.rs` Semigroup on RPN tuples for tree rollup.
**Does NOT solve**: runtime polymorphism without HKT; HList compile costs grow with length.

#### A2. rpds 1.2.0 + archery 1.2.2
**What it exposes** (rpds): `List`, `Vector` (RRB-tree), `Stack`, `Queue`, `HashTrieMap`/`HashTrieSet`, `RedBlackTreeMap`/`RedBlackTreeSet` — all O(log n) structural-sharing persistent. Each takes a `SharedPointerKind` from archery (`RcK`/`ArcK`).
**Where it lands in C3I**: `cortex.rs` ConversationHistory (50-msg sliding window) → `rpds::Vector<Message>` (ArcK) lock-free reads. `db.rs` SemanticCache hot tier → `HashTrieMap<QueryHash, CachedReply>`. `trace.rs` PipelineTracer.stages → `rpds::List` for forkable in-flight tracer for hedged inference. **Bridge**: Gleam side already uses persistent collections natively; rpds narrows the impedance mismatch at the NIF boundary.
**Does NOT solve**: zero-cost interior mutation; mutable iteration awkward; not faster than Vec for hot inner loops.

#### A3. winnow 1.0.2
**What it exposes**: `Parser` trait; combinators `alt`/`repeat`/`delimited`/`preceded`/`terminated`/`opt`/`cut_err`; `Stateful` for context; `ContextError`/`TreeError` for human-readable errors; `dispatch!` for token-discriminated branching.
**Where it lands in C3I**: `markdown.rs` (124 LOC) inverse parser — read PROJECT_TODOLIST.md back into Tasks (round-trip integrity check). `cli.rs` (261 LOC) dispatcher. `gemini_live.rs` (307 LOC) WebSocket frame parser. `cortex.rs` intent classifier first-stage (sub-1ms tier).

#### A4. tower 0.5.3
**What it exposes**: `Service<Request>` (≅ `Request → Future<Output=Result<Response, Error>>` = Kleisli); `Layer` (Service combinator); built-in `Timeout`/`Retry`/`RateLimit`/`LoadShed`/`Buffer`/`ConcurrencyLimit`/`Map*`/`Then`; `ServiceBuilder` fluent stacking.
**Where it lands in C3I**: `mcp_inference.rs` (663 LOC, 6-tier hedged) → `ServiceBuilder::new().layer(Timeout).layer(Retry).layer(CircuitBreaker).service(GeminiDirect)`. `gateway.rs` (198 LOC) → `tower::buffer::Buffer` per channel + `tower::Steer` for routing. `mcp_gworkspace.rs` SMTP/OAuth2 → `tower-retry` + `tower-rate-limit`.
**REJECT** for vault NIF and BEAM-side NIFs (sync only).

#### A5. derive_more 2.1.1
**What it exposes**: `#[derive(From, Into, AsRef, Deref, DerefMut, Display, Debug, Add, Sub, Mul, Sum, Product, Constructor, IsVariant, Unwrap, TryUnwrap)]`.
**Where it lands**: `types.rs` (850 LOC) — every `String` field that means something specific (`ChatId`, `IntentId`, `MeshNodeId`, `RunId`, `TaskUrn`) becomes derived newtype. `errors.rs` (226 LOC) — `derive_more::Display + From + Error` removes ~40% of file. `c3i_nif/src/planning.rs` boundary — newtypes prevent `priority`/`status` cross-typing. `rusty_vault_nif` — `SecretName`/`KekFingerprint`/`SealedBytes` as newtypes.

#### A6. nutype 0.7.0 — refinement types ("parse don't validate")
**What it exposes**: `#[nutype]` proc-macro accepting `validate(...)`, `sanitize(...)`, `derive(...)` clauses; produces a wrapper whose constructor is the **only** way to obtain it. Validators include `len_char_min/max`, `regex`, `predicate`, `greater`, `less`, `finite`, custom closures.
**Where it lands**: **Direct fit for SC-VALUE-GUARD-001..008.** Replace `&'static [&str]` whitelists with `#[nutype(validate(predicate = |s| matches!(s, "P0"|"P1"|"P2"|"P3")), derive(...))] pub struct Priority(String);` — now the L3 daemon literally cannot be called with invalid priority. Vault: `SecretName` constrained to `^[a-z][a-z0-9_-]{2,63}$`; `Ttl` constrained to `1m..=168h` per `secret_policy` table. **Highest-leverage adoption surface.**
**Does NOT solve**: validation depending on external state.

#### A7. recursion 0.5.4
**What it exposes**: `Collapsible`/`Expandable` traits; `MappableFrame` type-functor; cata/ana morphisms evaluated on the heap (no stack blowup).
**Where it lands**: `ruliology.rs` (929 LOC) — pure cata for `causal_cone()`, `lyapunov_lambda()`, `entropy_h()`. `rule_engine.rs` (961 LOC) RETE-UL alpha-network conflict-set traversal. `fmea.rs` RPN tree fold. `plan_search`/`plan_get` recursion through subtask hierarchy.

#### A8. proptest 1.11.0
**What it exposes**: `proptest!` macro; `Strategy` trait (composable random generator); shrinking; persistent failure files; `prop_assume!`.
**Where it lands**: `pii.rs` (91 LOC) — `scrub(scrub(x)) == scrub(x)` (idempotence), `scrub(x).contains_email == false`. `db.rs` `validate_priority`/`validate_status` — `∀ s ∈ VALID_STATUSES. validate(s).is_ok()`. `rule_engine.rs` 52 GRL rules — `∀ ctx. evaluate(ctx).is_some()` no-panic property. `trace.rs` PipelineTracer — `finish_with_zenoh` always emits exactly one TraceSummary. `gateway.rs` parallel broadcast — `∀ msg. broadcast(msg)` eventually-delivered (using `proptest-state-machine`).

#### A9. kani-verifier 0.67.0
**What it exposes**: `#[kani::proof]` attribute; `kani::any()`/`kani::assume()`/`kani::cover()`; bounded model checking with CBMC; SAT-solver-backed exhaustive verification within bounds.
