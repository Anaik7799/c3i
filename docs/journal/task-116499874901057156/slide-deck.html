Confirmed Ecosystem Gaps (Do Not Author)

Four real gaps in the Rust FP ecosystem, each with a recommended workaround:

  • Optics (lens-rs/pl-lens dead) → use frunk::LabelledGeneric + derive_more
  • Algebraic effects (effing-mad/eff hobbyist) → use tower::Layer + tracing
  • STM (stm crate from 2018, abandoned) → use SQLite WAL (SC-VAULT-012 already mandates this)
  • Free monads / tagless final → use closed enums + handlers (idiomatic Rust)
C3I gains more from leaning on existing constraint families (SC-WIRE, SC-VALUE-GUARD, SC-VAULT, SC-CPIG) than from authoring generic FP libraries.
