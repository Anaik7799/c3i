## 1. Scoring methodology

```
Composite = 2·C + 1.5·K + 1.2·U + 1.0·P − 0.5·E
  where each axis is scored 1..10
  U = Utility           (breadth of impact across the swarm)
  C = Cost savings      ($ per year savings potential)
  K = Criticality       (risk if NOT done)
  P = Performance       (latency + throughput improvement)
  E = Effort (inverse)  (more effort = more penalty)
```

## 5. Hidden-gem (under-scored but strategic)

1. **#19 Per-turn span publishing** — Correlate *which prompt* caused which cost. Enables surgical fixes instead of population-level tuning.
2. **#10 Shared cross-agent cache** — Claude pays cache-write; Pi pays again. Shared SQLite projection → second agent hits cache on first token. ~50 % cumulative win.
3. **#7 Exact-match prompt cache** — Dumb but devastating. Sessions re-send identical prompts (`run tests`) repeatedly.
4. **#13 MoE routing** — Cortex intent classifier already exists at `cortex.rs:232`. Just wire the decision to the provider.
5. **#15 Skill-scoped system prompts** — Pi loads all 44 tool schemas on every turn. Scoping to 6–10 active tools saves ~4 k tokens/turn.
