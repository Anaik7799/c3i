## 4. Root Cause Analysis

**Why shared building blocks matter**: Without them, Pi and Claude diverge silently. Pi validates 10/32 events while Claude validates 32/32. When Pi emits an event Claude doesn't recognize, or vice versa, the bridge drops it silently. Shared building blocks mean both sides validate identically.

**Why generate FROM source**: If Pi's types are hand-written, they drift from Gleam's types over time. By extracting directly from `events.gleam` line 10-44, we guarantee structural parity. Future maintenance: when Gleam adds event type #33, grep for `AGUI_EVENT_TYPES` in Pi and add it.

## 6. Patterns & Anti-Patterns Discovered

### Patterns (keep)
- **Generate from source**: Read Gleam/Rust → emit TypeScript. Never hand-write parallel types.
- **Static fallback + dynamic sync**: MCP registry has 73 static tools + optional daemon query. Works offline.
- **Per-tier isolation**: Each circuit breaker is independent. Gemini failure doesn't block Ollama.
- **Unified re-export**: `shared-building-blocks.ts` is the single import for all shared code.

### Anti-Patterns (avoided)
- **Hand-written parallel types** that drift from source.
- **Single global breaker** that cascades failure across all tiers.
- **Hardcoded tool lists** that fall behind as sa-plan-daemon evolves.
- **Keyword-based PII detection** that misses structured patterns (emails, IPs).
