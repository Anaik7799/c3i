02 — Problem

Tests existed, but the system couldn't see them

  • fluffychat had patrol: ^4.5.0 as dev_dep but no native runner.
  • Authoring required guessing widget keys (anti-pattern zk-3346fc607a1ef9e6).
  • Single-platform regression — Android-only or Web-only.
  • Test results lived in stdout. Cortex/dashboard had no visibility.
  • No bridge between AI assistants and the running Flutter app.
03 — Solution shape

Two MCP servers, one mesh

Patrol MCP — orchestrate

  • run · screenshot · native-tree
  • status · quit
  • Drives written tests across Android · Linux · Chrome

Marionette MCP — introspect

  • connect · get_interactive_elements
  • tap · enter_text · scroll_to
  • take_screenshots · get_logs · hot_reload
  • Live VM-Service-driven discovery in debug builds

Every tool call → Zenoh topic indrajaal/l5/test/{patrol,marionette}/<id>/<phase>

04 — System flow

OODA loop with MCP+Zenoh

Observe   → marionette.get_interactive_elements + patrol.native-tree
Orient    → ZK recall (UserPromptSubmit hook) + last test envelope on Zenoh
Decide    → write/edit patrol_test.dart with discovered selectors
Act       → patrol.run (publishes start, screenshots, status)
Verify    → /passed or /failed envelope; on fail, recurse with Marionette

Hook is async (5 s) so the model is never blocked.
