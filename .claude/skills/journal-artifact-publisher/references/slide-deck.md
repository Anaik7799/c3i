# Slide Deck Reference

Use this when drafting `*-deck.html`.

## Requirements

- HTML-only, scrollable without JavaScript.
- 8-14 slides for operator review.
- Each slide is one `<section>`.
- No external CSS, fonts, scripts, images, or CDN dependencies.
- Include task ID, bundle slug, and route status in the opening slide.

## Required Slide Topics

1. Title and task context.
2. Artifact inventory.
3. Task-management integration.
4. What changed.
5. Validation evidence.
6. Link/route state.
7. Risk register.
8. Git staging boundary.
9. Remaining gaps.
10. Decision / next actions.

## Avoid

- Speaker-note-only content that is invisible in the rendered deck.
- JavaScript keyboard handlers.
- Claims that a route or email worked unless verified.
