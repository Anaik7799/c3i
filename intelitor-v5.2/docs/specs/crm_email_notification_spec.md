# CRM Email Notification Service Specification

> Version: 21.3.1-SIL6 | Date: 2026-03-28 | STAMP: SC-MSG-001 to SC-MSG-006
> Task: bf29cb31 (P3-UI), related: 545b9f77

## Overview

The CRM Email Notification Service delivers alarm alerts, digest summaries,
and scheduled reports to operators via email. It uses Broadway for concurrent
message processing, EEx templates for rendering, and configurable delivery
channels. All email dispatch is gated by the policy engine and logged to
the immutable audit trail.

## Design Intent

- Deliver timely, actionable notifications to operators
- Support multiple email types with distinct urgency levels
- Render emails from reusable, branded templates
- Process email jobs concurrently via Broadway pipeline
- Audit all sent emails in the immutable register (SC-SAFETY-003)

## Architecture

```
Alarm / Scheduled Event
  |
  v
NotificationRouter (decides email type + recipients)
  |
  v
Broadway Pipeline (batched, concurrent)
  |
  +-- TemplateRenderer (EEx templates)
  +-- PolicyGate (SLA check, rate limit, opt-out)
  +-- DeliveryAdapter (SMTP / SendGrid / Mailgun)
  |
  v
Audit Logger --> Immutable Register
```

## Email Types

### 1. Alert Email (Immediate)

Triggered by alarms of severity Critical or High. Sent within 60 seconds of
alarm activation.

| Field | Value |
|-------|-------|
| Subject | `[CRITICAL] Alarm: {source} - {message}` |
| Priority | High (X-Priority: 1) |
| Recipients | On-call roster + site managers |
| Template | `alert.html.eex` |
| Rate Limit | Max 10 per source per hour |

### 2. Digest Email (Scheduled)

Aggregated summary of all alarms and events over a configurable period.
Default: daily at 08:00 CEST.

| Field | Value |
|-------|-------|
| Subject | `[Digest] Indrajaal Daily Summary - {date}` |
| Priority | Normal |
| Recipients | All subscribed operators |
| Template | `digest.html.eex` |
| Schedule | Configurable (daily/weekly/monthly) |

### 3. Report Email (On-Demand / Scheduled)

Attached report generated by the Analytics Report Builder. Supports PDF and
CSV attachments.

| Field | Value |
|-------|-------|
| Subject | `[Report] {report_name} - {date}` |
| Priority | Normal |
| Recipients | Report requester + distribution list |
| Template | `report.html.eex` |
| Attachments | PDF, CSV (max 10MB) |

## Template System

Templates use EEx (Embedded Elixir) with a shared layout.

### Template Directory

```
lib/indrajaal_web/templates/email/
  |-- layout.html.eex       Base layout (header, footer, branding)
  |-- alert.html.eex        Alert notification body
  |-- digest.html.eex       Daily/weekly digest body
  |-- report.html.eex       Report delivery body
  |-- components/
       |-- alarm_row.html.eex    Single alarm row component
       |-- metric_card.html.eex  KPI metric card component
       |-- footer.html.eex       Unsubscribe + contact footer
```

### Template Variables

| Variable | Type | Available In |
|----------|------|--------------|
| `@alarm` | map | alert |
| `@alarms` | list | digest |
| `@report` | map | report |
| `@recipient` | map | all |
| `@unsubscribe_url` | string | all |
| `@site_name` | string | all |
| `@generated_at` | DateTime | all |

## Broadway Pipeline

```elixir
defmodule Indrajaal.CRM.EmailPipeline do
  use Broadway

  @impl true
  def handle_message(:default, message, _context) do
    message
    |> render_template()
    |> check_policy()
    |> deliver()
    |> audit_log()
  end

  @impl true
  def handle_batch(:default, messages, _batch_info, _context) do
    # Batch delivery for digest emails (up to 50 per batch)
    messages
  end
end
```

### Pipeline Configuration

| Parameter | Value |
|-----------|-------|
| Concurrency | 4 processors |
| Batch size | 50 (digest), 1 (alert) |
| Batch timeout | 10 seconds |
| Rate limit | 100 emails/minute global |
| Retry | 3 attempts with exponential backoff |

## Audit Trail

Every email dispatch is recorded in the immutable register:

```json
{
  "event": "email_sent",
  "type": "alert",
  "recipient": "operator@example.com",
  "subject": "[CRITICAL] Alarm: sensor-bay-2 - temperature_high",
  "alarm_id": "alm-20260328-001",
  "delivered_at": "2026-03-28T14:30:00Z",
  "delivery_status": "accepted",
  "adapter": "sendgrid"
}
```

## Error Handling

| Error | Action |
|-------|--------|
| SMTP connection failure | Retry 3x, then queue for next batch |
| Template rendering error | Log error, send plain-text fallback |
| Rate limit exceeded | Queue message, deliver in next window |
| Invalid recipient | Log warning, skip, notify admin |
| Attachment too large | Truncate or link to download URL |
